Схема данных:

Таблица videos:
- id (идентификатор видео)
- video_created_at (дата и время ПУБЛИКАЦИИ видео)
- views_count
- likes_count
- comments_count
- reports_count
- creator_id (идентификатор креатора)
- created_at (дата и время СОЗДАНИЯ видео)
- updated_at

Таблица video_snapshots (замеры раз в час):
- id (идентификатор снапшота)
- video_id (FOREIGN KEY на соответствующее видео)
- views_count
- likes_count
- comments_count
- reports_count
- delta_views_count
- delta_likes_count
- delta_comments_count
- delta_reports_count
- created_at (дата и время замера)
- updated_at

Примечания к схеме:
- videos.*_count (финальное значение)
- video_snapshots.*_count (значение на момент замера)
- video_snapshots.delta_*_count (приращение с прошлого замера)

Подсказки:
- "сколько видео" -> COUNT(DISTINCT videos.id)
- "выросли просмотры" -> SUM(delta_views_count)
- "получали новые просмотры" -> delta_views_count > 0
- "финальное количество лайков" -> videos.views_count
- "видео опубликовано в ноябре" -> video_created_at
- "видео создано" -> created_at

Пример ответа:
SELECT SUM(delta_views_count) AS views_count
FROM video_snapshots
WHERE delta_views_count > 0;
