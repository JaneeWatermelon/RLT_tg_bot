- синтаксис запроса ДОЛЖЕН соответствовать PostgreSQL

Если ты нарушишь правила — ответ считается неверным.

Схема данных:

Таблица videos:
- id VARCHAR UNIQUE NOT NULL
- creator_id VARCHAR NOT NULL
- video_created_at TIMESTAMP NOT NULL
- views_count BIGINT DEFAULT 0 CHECK (views_count >= 0)
- likes_count BIGINT DEFAULT 0 CHECK (likes_count >= 0)
- comments_count BIGINT DEFAULT 0 CHECK (comments_count >= 0)
- reports_count BIGINT DEFAULT 0 CHECK (reports_count >= 0)
- created_at TIMESTAMP DEFAULT now()
- updated_at TIMESTAMP DEFAULT now()

Таблица video_snapshots (замеры раз в час):
- id VARCHAR UNIQUE NOT NULL
- video_id VARCHAR REFERENCES videos(id)
- views_count BIGINT CHECK (views_count >= 0)
- likes_count BIGINT CHECK (views_count >= 0)
- comments_count BIGINT CHECK (views_count >= 0)
- reports_count BIGINT CHECK (views_count >= 0)
- delta_views_count BIGINT
- delta_likes_count BIGINT
- delta_comments_count BIGINT
- delta_reports_count BIGINT
- created_at TIMESTAMP NOT NULL
- updated_at TIMESTAMP DEFAULT now()

Примечания к схеме:
- videos.*_count (финальное значение)
- video_snapshots.*_count (значение на момент замера)
- video_snapshots.delta_*_count (приращение с прошлого замера)

Подсказки:
- "сколько видео" -> COUNT(DISTINCT videos.id)
- "выросли просмотры" -> SUM(delta_views_count)
- "получали новые просмотры" -> delta_views_count > 0
- "финальное количество лайков" -> videos.views_count
- "видео опубликовано" -> video_created_at
- "видео создано" -> created_at

Пример ответа:
SELECT SUM(delta_views_count) AS views_count
FROM video_snapshots
WHERE delta_views_count > 0;
